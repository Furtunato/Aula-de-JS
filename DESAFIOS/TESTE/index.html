Levantamento de Requisitos - Pizzaria

Requisitos Funcionais

ID Descrição
RF01 O sistema deve permitir o cadastro de clientes.
RF02 O sistema deve permitir o cadastro de pedidos.
RF03 O sistema deve permitir o cadastro de produtos (pizzas, bebidas, etc.).
RF04 O sistema deve gerar relatórios de vendas.

Requisitos Não Funcionais - Facilidade de Uso, Eficiência, Desempenho, Espaço, Confiabilidade e Portabilidade

ID Descrição
RNF01 O sistema deve ser intuitivo e fácil de usar.
RNF02 O sistema deve responder às ações do usuário em até 2 segundos.
RNF03 O sistema deve suportar até 1000 pedidos simultâneos.
RNF04 O sistema deve ocupar no máximo 500MB de espaço em disco.
RNF05 O sistema deve ter uma disponibilidade de 99,9%.
RNF06 O sistema deve ser compatível com Windows, macOS e Linux.

Requisitos Não Funcionais - Entrega, Implementação e Padrões

ID Descrição
RNF07 O sistema deve ser entregue em até 6 meses.
RNF08 O sistema deve ser implementado em fases.
RNF09 O sistema deve seguir os padrões de desenvolvimento web W3C.

Requisitos Não Funcionais - Interoperabilidade, Éticos, Legais, Privacidade e Segurança

ID Descrição
RNF10 O sistema deve ser capaz de se integrar com sistemas de pagamento online.
RNF11 O sistema deve respeitar os princípios éticos de uso de dados.
RNF12 O sistema deve estar em conformidade com a legislação vigente.
RNF13 O sistema deve garantir a privacidade dos dados dos usuários.
RNF14 O sistema deve implementar medidas de segurança contra ataques cibernéticos.
